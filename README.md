# Go Do Matcher Bot

Telegram-бот хранит только минимальные данные пользователя и его поисковый запрос.
Тексты вакансий не сохраняются: новая вакансия из канала сразу матчится с активными пользователями.

## Хранение данных

Бот использует PostgreSQL из переменной `DATABASE_URL`.

Таблицы создаются автоматически при старте:

- `bot_users` - пользователь Telegram, его имя, запрос, статус подписки.
- `sent_vacancies` - отметки, какая вакансия уже отправлена конкретному пользователю.

## Переменные окружения

```env
BOT_TOKEN=
CHANNEL_USERNAME=goDoMinsk
DATABASE_URL=
OPENAI_API_KEY=
OPENAI_TRANSCRIBE_MODEL=gpt-4o-mini-transcribe
PUBLIC_BASE_URL=
WEBHOOK_URL=
```

`OPENAI_API_KEY` нужен только для голосовых запросов. Если ключ не задан, текстовые запросы продолжат работать.

## Миграция старого JSON

Старый файл `data/bot.json` можно перенести один раз:

```bash
go run ./cmd/migrate-json ./data/bot.json
```

Миграция переносит пользователей и отметки отправленных вакансий. История текстов вакансий намеренно не переносится.
